Для создания платёжной операции необходимо отправить запрос к API Finmart. 
В ответ вы получите ссылку на платёжную страницу и идентификатор операции.

## Обязательные параметры создания

- `method` - метод оплаты (`card`, `openbanking`, `applepay`, `googlepay`,`apm`, `widget_integration`, `cashbox` и др.)
- `payment_profile` - платёжный профиль. Обычно это `production` или `sandbox`, однако в ряде случаев возможны индивидуальные профили.
- `amount.amount` - сумма операции.
- `amount.currency` - валюта операции.
- `user.user_id` - идентификатор пользователя.



## Примеры запросов
  
 ### Банковская карта HPP
    ```bash
    curl --location 'https://sandbox.finmar.tech/prepare2/api/v3/checkout' \
    --header 'Content-Type: application/json' \
    --header 'Authorization: Basic cGxheWJybzptcTRWIWllNXFYaEE=' \
    --data '{
        "payment_profile": "sandbox",
        "method": "card",
         "client_operation_id": "Fz022tCSaBuvvfQckp",
        "user": {
            "user_id": "42d555d0-bdcb-4b47-a34c-731e3e91a2d0"
        },
        "amount": {
            "amount": "9",
            "currency": "EUR"
        },
        "url_callback": "https://gg.requestcatcher.com?callback",
        "url_redirect_success": "https://gg.requestcatcher.com?success",
        "url_redirect_fail": "https://gg.requestcatcher.com?fail",
        "merchant_user_entry_point_url": "mechant.site/cashier/deposit"
    }'
    ```

   ```bash
        "url_redirect_fail": "https://gg.requestcatcher.com?fail",
        "merchant_user_entry_point_url": "https://mechant.site/cashier/deposit"
    }'
    ```


В ответе на запрос вы получите:
- `success` - true/false. Обязательный параметр.
- `result.url` - Сссылка на платёжную страницу. Обязательный параметр в случае успешного создания операции. 
- `result.launch_type` - опциональный параметр. Рекомендуемый способ показа платёжной страницы пользователю.
- `kyc_status` - опциональный параметр. Наличие данных KYC пользователя.
- `user_stats` - опциональный параметр. Наличие статистических данных пользователя.

 ## Важные моменты
- Используйте уникальный `client_operation_id` для каждой операции
- Сохраняйте `reference_id` для дальнейшей работы с операцией
- Убедитесь, что URL для `url_callback` и `url_redirect_success`/ `url_redirect_fail` доступны и корректны.
- Не выполняйте запрос к API прямо из браузера в продакшене — используйте ваш сервер как прокси
