---
title: "Получение состояния выплаты"
---

```text
POST /public/api/withdraw/generic/v2/get_operation_state
```

С помощью этого метода можно в любой момент проверить состояние выплаты.

Запрос:

```json
{
  "client_operation_id": "93ca1dac-7479-42e6-b0b3-e721572fdc0f"
}
```

## Пример зависшей выплаты

В некоторых исключительных ситуациях исполнение выплаты может быть приостановлено (при нарушении инвариантов, продолжение автоматической работы приведет к непредсказуемым результатам). Пример такой ситуации ниже

```json
{
  "success": true,
  "trace_id": "05a38b61a9985fcbd1c5a026c5260882",
  "result": {
    "operation_state": {
      "operation_id": "01903af45f5e720db0656a8e2006dddd",
      "project_id": "check",
      "client_operation_id": "0f82bd9a-69cb-4677-9c21-0da1e8759e09",
      "reference_id": "01903af45f607272a2584e921e9c721a",
      "created_at": "2024-06-21T13:20:12.125Z",
      "updated_at": "2024-06-21T13:21:03.886Z",
      "version": 3,
      "order_amount": {
        "currency": "EUR",
        "amount": "50.99"
      },
      "explicit_liquidity_source_id": "some_eur",
      "recipient_payment_method": {
        "bank_card": {
          // токенизированная карта
          "pan_token": "pan_jiqt8hefrimm6ouaa64rxnx4n60l3dhu",
          "card_token": "card_ltq5lkp0gzfqnx660foqrutls8kdr0f3",
          // с частичным раскрытием данных
          "bin6": "444444",
          "last_four_digits": "0008",
          "expiration_month": "03",
          "expiration_year": "2033",
          // и расширенной информацией при ее доступности
          "network": "Visa",
          "country": "PL",
          "issuer": "CREDIT AGRICOLE BANK POLSKA S.A."
        }
      },
      "url_callback": "https://devnull-as-a-service.com/dev/null",
      "description": "some order description",
      "metadata": "arbitrary_data",
      "report_metadata": "billing_id_fe9057dbd7644c5cbe01ac3d454d6570",
      "disallow_end_user_interaction": true,
      "disallow_transaction_split": true,

      "execution_suspended": true
    }
  }

```

В случае получения такого состояния следует обратиться в службу поддержки. Исполнение выплаты продолжится только после ручного разбирательства.
