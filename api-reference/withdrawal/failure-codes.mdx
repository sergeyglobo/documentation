---
title: "Коды ошибок выплаты"
---

В процессе подготовки, запуска или исполнения выплаты операция может завершиться неуспешно. В этом случае операция содержит объект `failure_details` с описанием причины.

Пример объекта `failure_details`:

```json
{
  "failure_details": {
      "code": "PROVIDER_PROCESSING_ERROR",
      "details": "Transaction processing failed",
      "raw_provider_error_code": "fail_reason=ISSUER_ERROR_OTHER"
    }
}
```

Ниже приведён список возможных кодов ошибок и их общее описание.

| code | описание |
| --- | --- |
| `NO_SUITABLE_METHODS` | Не удалось подобрать подходящий способ исполнения выплаты. Все доступные варианты не прошли проверку условий выполнения. |
| `INSUFFICIENT_BALANCE` | Недостаточно средств для выполнения выплаты. Некоторые варианты исполнения прошли проверки, однако ни на одном из доступных субсчетов не оказалось достаточного баланса. |
| `WITHDRAWAL_START_TIMEOUT` | Выплата не была запущена в установленное время и была автоматически отклонена. |
| `ABORTED_BY_REQUEST` | Выплата была отменена по явному запросу клиента или в результате ручной корректировки. |
| `UNSUPPORTED_WITHDRAWAL_PARAMETERS` | Переданные параметры выплаты не поддерживаются в текущем контексте. Обычно означает, что для выбранного метода выплаты или профиля требуется другой набор параметров. |
| `INTERNAL_ERROR` | Внутренняя ошибка сервиса, не связанная с непосредственным исполнением выплаты. |
| `PROVIDER_PROCESSING_ERROR` | Ошибка при исполнении выплаты у внешнего платёжного провайдера. Детализация ошибки от провайдера передаётся в дополнительных полях 'raw\_provider\_error\_code' и 'raw\_provider\_error\_message'. |
| `UNSPECIFIED` | Причина ошибки не была классифицирована. Такое возможно, если операция была завершена через устаревшее Card API. |
