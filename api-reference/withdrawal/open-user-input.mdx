---
title: "Открытие интерфейса для конечного пользователя"
---

При переходе выплаты в состояние `USER_INPUT` партнеру необходимо перенаправить конечного пользователя на страницу сервис. Для получения адреса страницы необходимо вызвать метод

```text
POST /public/api/withdraw/generic/v2/open_operation_user_input
```

с телом:

```json
{
  "client_operation_id": "c806d5d6-11ec-4d1b-86f0-245247e0611c",

  // опциональный адрес для возвращения пользователя после завершения USER_INPUT
  "url_redirect": "https://my-awesome-project.com/return/url"
}
```

В случае успешного исполнения вернется:

```json
{
  "success": true,
  "trace_id": "9fc1399cee071523b4d8ae811b9742e3",
  "result": {
    "operation_state": {
      // текущее состояние операции
    },
    // параметры для обработки USER_INPUT
    "user_input_parameters": {
      // на этот адрес необходимо перенаправить пользователя
      "url": "https://service.domain/path/to/user/input",
      // это значение меняется при каждом переходе PROCESSING -> USER_INPUT
      "concurrency_stamp": "1f01e29240c84da8b7ca57860e127ab0",
      // время перехода в состояние USER_INPUT
      "requested_at": "2024-06-21T15:56:15.123Z",
      // текущий адрес перенаправления пользователя после завершения USER_INPUT
      "url_redirect": "https://my-awesome-project.com/return/url"
    }
  }
}
```

Возможны несколько специальных случаев ошибки:

- `OPERATION_NOT_FOUND`: выплата с переданным `client_operation_id` не существует на стороне сервиса
- `UNACCEPTABLE_COMMAND`: метод не может быть вызван в текущем состоянии выплаты, необходимо проверить актуальное состояние

Если при вызове метода был передан `url_redirect`, то после завершения `USER_INPUT` пользователь будет перенаправлен на этот адрес. В противном случае пользователь останется на финальной странице сервиса.

При повторных вызовах метода значение `url_redirect` перезаписывается, отсутствие параметра или явная передача `null` сбрасывает значение в `null` на стороне сервиса.
