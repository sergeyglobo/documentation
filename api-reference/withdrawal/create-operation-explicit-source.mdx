---
title: "Создание выплаты с явным указанием источника ликвидности"
---

```text
POST /public/api/withdraw/generic/v2/create_operation_using_explicit_source
```

Запрос создания выплаты имеет множество параметров:

```json
{
  // идентификатор операции на вызывающей стороне
  "client_operation_id": "string",
  // информация о конечном пользователе
  "user": {
    "user_id": "u1",
    "email": "user@example.domain",
    "email_verified": true
  },
  "recipient_payment_method": {
    // данные банковской карты на которую должны быть выплачены средства
    "bank_card": {
      "pan": "4444 4444 4444 0008",
      "cardholder": "JOHN SMITH",
      "expiration_month": 3,
      "expiration_year": 2033
    }
  },
  // явно переданный идентификатор источника ликвидности
  "liquidity_source_id": "some_eur",
  // сумма выплаты
  "order_amount": {
    "currency": "EUR",
    "amount": "50.99"
  },
  // адрес, на который необходимо отправлять уведомления о смене состояния
  "url_callback": "https://devnull-as-a-service.com/dev/null",
  // произвольное описание назначения выплаты
  "description": "some order description",
  // произвольные данные, которые будут доступны в состоянии выплаты
  "metadata": "arbitrary_data",
  // произвольные данные, которые попадут в финансовую отчетность
  // (может помочь при клиринге)
  "report_metadata": "billing_id_fe9057dbd7644c5cbe01ac3d454d6570",
  // запрет на переход в состояние USER_INPUT
  "disallow_end_user_interaction": true,
  // запрет разбивки выплаты на несколько отдельных переводов
  "disallow_transaction_split": true,
  // синхронная обработка состояния PREPARING, ответ на запрос будет возвращен только после перехода в START_WAIT/UNPROCESSABLE
  "await_preparing": false
}
```

В случае неуспешного вызова возможны следующие особенные ситуации:

- `AMBIGUOUS_PARAMETERS`: получен запрос с ранее обработанным `client_operation_id`, но отличающимися _другими параметрами_, в этом случае необходимо проверить текущее состояние выплаты по `client_operation_id`
- `SOURCE_NOT_FOUND`: переданный источник ликвидности не существует или недоступен, необходимо проверить логику выбора источника
- `UNSUPPORTED_CURRENCY`: источник ликвидности не поддерживает переданную валюту

При создании выплаты следует обратить внимание на feature flags:

- `disallow_end_user_interaction`: при установке в `true` гарантированно не возникнет состояние `USER_INPUT`. По умолчанию стоит значение `false`, при котором состояние `USER_INPUT` _может_ возникнуть и должно быть правильно обработано на стороне партнера. Установка значения `true` _может_ приводить к неуспешным результатам выплаты в случаях, когда использование `USER_INPUT` помогло бы ее исполнить успешно.
- `disallow_transaction_split`: при установке в `true` гарантированно исполняет выплату с помощью не более чем одной транзакции перевода на карту. По умолчанию стоит значение `false`, при котором сервис _может_ выполнить оптимизацию и разбить выплату на несколько транзакций. По аналогии с предыдущим флагом, значение `true` может повысить вероятность неуспешного результата исполнения выплаты.
- `await_preparing`: при установке в `true` обработка состояния `PREPARING` будет произведена синхронно внутри обработки запроса создания и ответ на этот запрос будет возвращен только после перехода в состояние `START_WAIT` или `UNPROCESSABLE`. Этот режим позволяет не использовать опрос состояния/обработку уведомлений для проверки возможности запуска выплаты - при получении ответа с `START_WAIT` можно сразу запустить выплату на исполнение.
