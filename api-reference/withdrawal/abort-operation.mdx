---
title: "Отмена выплаты"
---

```text
POST /public/api/withdraw/generic/v2/abort_operation
```

В некоторых ситуациях выплату можно отменить: если в настоящий момент нет незавершенного движения средств по ней.

Запрос:

```json
{
  "client_operation_id": "3c1a9b19-02d4-4768-846c-cd3996bc0100"
}
```

В случае успеха выплата перейдет в состояние `COMPLETE`, а `result` станет `FAILURE` (если движения средств не было) или `PARTIAL_SUCCESS` если выплата успела частично исполниться на момент отмены.

Возможны несколько специальных случаев ошибки:

- `OPERATION_NOT_FOUND`: выплата с переданным `client_operation_id` не существует на стороне сервиса
- `UNACCEPTABLE_COMMAND`: метод не может быть вызван в текущем состоянии выплаты, необходимо проверить актуальное состояние
- `ACCEPTED_WITHOUT_OBLIGATIONS`: отменить выплату прямо сейчас невозможно (например, уже отправлена заявка на выплату в банк), но в будущем возможно отменить получится если текущая работа над выплатой не приведет к ее успешному завершению. В этом случае сервис самостоятельно попытается отменить выплату _если у него появится такая возможность_. Отмена при этом не гарантируется.
