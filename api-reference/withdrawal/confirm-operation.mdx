---
title: "Подтверждение завершения выплаты"
---

```text
POST /public/api/withdraw/generic/v2/confirm_operation
```

При завершении выплаты, т.е. при ее переходе в состояние `COMPLETE`, партнер должен произвести необходимые действия на своей стороне, **после чего** подтвердить что работа с выплатой полностью завершена.

Запрос:

```json
{
  "client_operation_id": "93ca1dac-7479-42e6-b0b3-e721572fdc0f",

  // идентификатор подтверждения, может использоваться при клиринге
  "operation_ack_id": "0462bb98-ebe0-4fd3-bea2-d84185810cba"
}
```

Возможны несколько специальных случаев ошибки:

- `OPERATION_NOT_FOUND`: выплата с переданным `client_operation_id` не существует на стороне сервиса
- `UNACCEPTABLE_COMMAND`: метод не может быть вызван в текущем состоянии выплаты, необходимо проверить актуальное состояние
