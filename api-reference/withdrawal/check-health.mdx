---
title: "Проверка доступности сервиса"
---

```text
POST /public/api/withdraw/generic/v2/check_health
```

Этот метод можно использовать не только для проверки корректности учетных данных, но и для проверки работоспособности самого сервиса.

Тело запроса не содержит каких-либо специальных параметров:

```json
{}
```

В ответе возможны 2 ситуации. Все хорошо:

```json
{
  "success": true,
  "trace_id": "d42d479e595a284b6034f5e21c49ab74"
}
```

Или имеются некоторые проблемы:

```json
{
  "success": false,
  "trace_id": "902722e84091b553873f105b1c4c2d96",
  "error": {
    "code": "UNAUTHORIZED",
    "details": "authentication failed"
  }
}
```

В случае повторяющихся проблем следует обратиться в поддержку.
