## Аутентификация и авторизация[​](#аутентификация-и-авторизация "Прямая ссылка на Аутентификация и авторизация")

Аутентификация в API может выполняться различными способами, клиент сам выбирает между простотой интеграцией и безопасностью. Необходимые для аутентификации данные передаются в HTTP заголовках.

Для каждой интеграции одновременно может использоваться несколько наборов учетных данных, это может применяться для их ротации или миграции на другую схему аутентификации.

К каждому набору учетных данных могут быть добавлены ограничения, например, белый список IP адресов, доступные методы API, настройки платежных методов и пр.

### Basic[​](#basic "Прямая ссылка на Basic")

Стандартная [Basic HTTP authentication scheme (RFC 7617)](https://datatracker.ietf.org/doc/html/rfc7617), дополнительная проверка подлинности отправителя запроса **не выполняется** - учетные данные полностью передаются в запросе. Эта схема предназначена для ускорения процесса интеграции.

Для клиента создается пара `key` + `password`, которые он передает в заголовке `Authorization`:

```
Authorization: Basic base64(key:password)
```
