### Коды ошибок операций

Эти коды передаются в callback-уведомлениях (поле `payload.card.reason_code` или аналогичное, в зависимости от провайдера, но унифицированные статусы доступны в `current_status` и деталях).

| Код | Описание |
|-----|----------|
| `SUCCESS` | Операция проведена успешно |
| `PROCESSING_INSUFFICIENT_FUNDS` | Недостаточно средств на карте |
| `PROCESSING_3D_SECURE_FAILED` | Ошибка 3D Secure подтверждения |
| `PROCESSING_3D_SECURE_CANCELLED` | 3D Secure отменён пользователем |
| `PROCESSING_DECLINED` | Операция отклонена процессингом (общий отказ) |
| `PROCESSING_TIMEOUT` | Превышено время ожидания ответа от банка |
| `PROCESSING_ERROR` | Общая техническая ошибка процессинга |

### Статусы операций

| Статус | Описание |
|--------|----------|
| `user_input_required` | Ожидание ввода данных пользователем |
| `in_process` | Операция в процессе обработки |
| `success` | Операция завершена успешно |
| `fail` | Операция завершена с ошибкой |

### HTTP-статусы API

| Статус | Описание |
|--------|----------|
| `200`  | Успешный запрос |
| `400`  | Ошибка валидации (проверьте параметры) |
| `401`  | Ошибка авторизации (проверьте логин/пароль) |
| `404`  | Эндпоинт не найден |
| `500`  | Внутренняя ошибка сервера |

<Tip>
  Для тестирования различных кодов ошибок используйте [тестовые карты](/api-reference/integration/checkout#тестовые-карты) с соответствующими CVV-кодами или суммами.
</Tip>
