Перед запуском приема платежей по картам в продакшн убедитесь, что все пункты выполнены.

<Steps>
  <Step title="Создание операции">
    - [ ] Операция создается успешным запросом (HTTP 200).
    - [ ] Передаются все обязательные параметры (`amount`, `currency`, `user_id`).
    - [ ] `client_operation_id` уникален для каждой попытки.
  </Step>
  
  <Step title="Перенаправление">
    - [ ] Пользователь корректно перенаправляется на платёжную страницу.
    - [ ] Обрабатываются `url_redirect_success` и `url_redirect_fail`.
    - [ ] Если используется iframe/new_window, поведение соответствует `launch_type`.
  </Step>
  
  <Step title="Обработка уведомлений (Callbacks)">
    - [ ] Сервер принимает POST-запросы на `url_callback`.
    - [ ] Проверяется цифровая подпись уведомления.
    - [ ] Сервер отвечает `200 OK` или `204 No Content` на валидные уведомления.
    - [ ] Уведомления обрабатываются идемпотентно (повторный колбек с тем же ID не вызывает дублирования зачисления).
  </Step>
  
  <Step title="Подтверждение и сверка">
    - [ ] Реализовано подтверждение успешной операции (API `client_confirm_payment`).
    - [ ] Настроен периодический опрос неподтверждённых операций (`client_unconfirmed_success_payments`).
  </Step>
  
  <Step title="Ошибки">
    - [ ] Корректно обрабатываются статусы `fail` и коды отказов (например, недостаток средств).
    - [ ] Пользователю показывается понятное сообщение об ошибке.
  </Step>
</Steps>

<Warning>
  Не запускайте интеграцию в продакшн с тестовыми кредами!
</Warning>
