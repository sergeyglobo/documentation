The **Withdrawal Card API v2** lets you automate payouts to bank cards through Vimapay.  
This section is generated from the `withdrawal_api_v2.1.json` OpenAPI specification and provides an interactive reference for all endpoints.

<Note>
All endpoint details (parameters, responses, and error models) are taken directly from the `withdrawal_api_v2.1.json` schema and stay in sync with the actual API.
</Note>

## When to use the Withdrawal Card API

Use this API when you need to:

- Create and control **card withdrawal operations**
- Check **service health** and **merchant balance**
- Manage the **operation lifecycle**: create → start → user input → confirm / abort → final status

The base URL for all requests in this API:

```bash
https://crypto.sopogai.com
```

## Basic flow

Typical card payout integration follows this pattern:

<Steps>
<Step title="Check service health and balance">
Use `/public/api/withdraw/card/v2/check_health` to ensure the service is available,  
and `/public/api/withdraw/card/v2/get_balance` to check that you have sufficient available balance.
</Step>

<Step title="Create a withdrawal operation">
Create a withdrawal operation with card data, amount, and user info using  
`/public/api/withdraw/card/v2/create_operation` or  
`/public/api/withdraw/card/v2/create_operation_using_explicit_source`.
</Step>

<Step title="Start the operation">
Call `/public/api/withdraw/card/v2/start_operation` to actually start processing.
</Step>

<Step title="Handle user interaction (optional)">
If user interaction is required (3‑DS, additional checks, etc.), use  
`/public/api/withdraw/card/v2/open_operation_user_input` to obtain a URL and redirect the user.
</Step>

<Step title="Track and finalize the operation">
Poll `/public/api/withdraw/card/v2/get_operation_state` to track status,  
optionally abort via `/public/api/withdraw/card/v2/abort_operation`,  
and confirm completion with `/public/api/withdraw/card/v2/confirm_operation`.
</Step>
</Steps>
